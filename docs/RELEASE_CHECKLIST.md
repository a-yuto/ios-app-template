# リリース準備チェックリスト

App Store 申請までに必要な、コード外の作業の順序付きチェックリスト。
リードタイムが長いもの (銀行確認・アカウント審査など最大24h) から先に着手すること。

実体験ベース: niki-sandbox#103 (loop-practice の申請で実際に踏んだ全項目)。

## 1. ASC / 外部サービス (リードタイム長い順)

- [ ] Apple Developer Program **使用許諾契約の同意状態を確認** (更新があると提出がブロックされる)
- [ ] **有料アプリ契約** (サブスク / 有料アプリの場合のみ):
  法人情報確認 → 署名 → 銀行口座 → 納税フォームの順。
  W-8BEN は Title=Individual。銀行確認に最大 24h かかる
- [ ] **ASC アプリレコード作成**。
  ⚠️ Bundle ID の選択ミスに注意 — 旧 ID を選ぶとビルドアップロードで詰む
- [ ] サブスク商品を **Ready to Submit** に (価格・ローカライズ・審査用スクショ・審査メモ)
- [ ] AdMob 登録 → **本番 App ID / 広告ユニット ID** を取得 (アカウント審査 〜24h)
- [ ] **プライバシーポリシー / 利用規約の公開 URL** を用意
  (解約手順は Apple サポートページへのリンクにすると UI 変更に耐性がある)

## 2. ASC メタデータ

- [ ] 説明文・プロモテキスト
  ⚠️ プロモーションテキスト欄は「→」が無効文字で保存できない
- [ ] サブタイトル / キーワード / サポート URL / 著作権
- [ ] **App Privacy 栄養ラベル**。AdMob 構成の場合:
  - 位置情報(おおよそ)・デバイス ID・製品の操作・広告データ = サードパーティ広告
  - 診断 3 種 = アプリの機能
  - すべて「ユーザに紐付けない」「トラッキングに使用しない」
- [ ] 年齢制限指定 (広告ありのアプリは「広告」だけ「はい」)
- [ ] 価格 (無料) と配信国
  (日本のみにすると EU DSA のトレーダー要件を回避できる)
- [ ] スクリーンショット (6.5" = 1284×2778)
  - `bundle exec fastlane screenshots` で撮影 (ScreenshotCaptureUITests 参照)
  - シミュレータ出力 1320×2868 からは `sips -z 2778 1284` でリサイズ可

## 3. コード側

- [ ] **AppIcon** (1024 PNG。無いとアーカイブ検証で必ず弾かれる)。
  プレースホルダーからの差し替え: `scripts/generate-app-icon.sh` 参照
- [ ] `ITSAppUsesNonExemptEncryption=false` (テンプレートの `Config/Info.plist` に同梱済み)
- [ ] **本番 ID 差し替え** (AdMob App ID / 広告ユニット ID など) +
  プレースホルダー混入防止テストを書く
- [ ] 規約 / ポリシー URL の差し替え (`example.com` のままはリジェクト定番)
- [ ] `useProductionAds` 等の本番フラグは**提出ビルド直前**に切り替える
  (ドッグフーディング中に本番広告を踏むとポリシー違反)

## 4. 配信

- [ ] ASC API キー作成 → `bundle exec fastlane beta` で TestFlight へ
  (キーの作り方は [fastlane/README.md](../fastlane/README.md) 参照、niki-sandbox#96)
- [ ] 実機検証
- [ ] 審査提出
